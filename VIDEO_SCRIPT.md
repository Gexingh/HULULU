# 呼噜噜 TruthLens｜2 分钟演示脚本

## 0:00–0:15 开场

大家好，这是呼噜噜 TruthLens，一个基于 Gonka 去中心化推理网络的公共信息核验工具。面对一条真假难辨的网络传言，它不会只问一个 AI，而是让两个 AI 独立对质。

## 0:15–0:35 输入案例

这里以“长城是唯一能从月球上用肉眼看到的人造建筑”为例。用户可以直接粘贴文本，也可以附上公开网页链接，让系统实时读取原文。现在点击开始核验。

## 0:35–1:10 展示结果

系统通过 Gonka Router 同时调用 Kimi-K2.6 和 MiniMax-M2.7。Kimi 作为事实调查员，负责拆解主张、检查证据；MiniMax 作为反方审计员，主动寻找反例、来源问题和不确定性。

两个模型独立给出分数后，系统使用公开公式计算最终 Truth Score，并对模型分歧进行扣分。这个案例被判断为高风险误导。

## 1:10–1:35 展示可追溯性

向下可以看到两个模型各自的结论、关键判断、参考证据和不确定性。每份报告底部都展示 Gonka API 实际返回的 Request ID，证明所有推理均通过 Gonka 网络完成，而不是页面生成的模拟结果。

## 1:35–1:55 技术与价值

项目没有调用其他大模型 API，也不会保存用户输入。它希望用低成本、多模型、可追溯的方式，帮助普通中文用户在转发之前多做一次判断。

## 1:55–2:05 结尾

呼噜噜 TruthLens——别急着转发，先让两个 AI 对质。谢谢。

## 录制提示

- 浏览器缩放建议 90%，保证评分卡和 Request ID 清晰可见。
- 提前跑好一次结果，录制时可展示已有结果，避免等待模型响应占用视频时长。
- 录屏需要完整出现在线网址、产品输入、Truth Score、双模型报告和 Request ID。
- 最后停留在 Request ID 区域 3 秒。
